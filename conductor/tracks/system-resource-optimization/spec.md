# Specification: 群晖系统资源占用优化、空闲写盘压制与安全漏洞修复

## 1. 业务需求 (Requirements)
1. **解决高频写盘与群晖休眠问题**：原 Docker 在空闲时因 MongoDB 预写日志及网络连接日志触发高频磁盘写入，导致磁盘几乎一直在读写。需在不影响数据安全的前提下极大降低写入动作频次。
2. **极大降低本地服务器 CPU 与带宽开销**：原影视流 TS 切片全部经由本地 Node.js 代理服务器中转，造成公网出口带宽和群晖 CPU 负载极高。需优化切片直连协议，最大化卸载服务器流量压力。
3. **内存控制与防泄漏**：限制 MongoDB Docker 的最大缓存空间，并在前端及时销毁视频播放器及其相关监听器，根治卸载组件后的内存堆积。
4. **安全加固**：修补 `image-proxy` 等对外服务接口的路径穿越（Directory Traversal）漏洞，防范本地任意敏感文件泄露风险。

## 2. 技术设计方案 (Technical Design)

### 2.1 修复安全与内存泄漏 (Commit: 8da6fa7)
* **路径穿越防御**：在 `app/api/image-proxy/route.ts` 中引入 URL 解析与白名单校验，强制解密后路径仅限安全域名及本地图片，拦截 `../` 等越权路径。
* **播放器销毁逻辑**：在播放页面卸载组件的生命周期钩子中，显示调用 Artplayer 实例的 `.destroy()` 方法，清除绑定在全局 window/document 的监听，避免垃圾回收（GC）无法释放播放器内存。

### 2.2 压制空闲写盘频次 (Commit: 9514e13)
* **MongoDB 日志与持久化优化**：
  * 在群晖部署中，修改 MongoDB 的配置启动命令，禁用预写 Journal (`--nojournal`) 以降低单条细微改动的实时写盘频次。
  * 调整 MongoDB 日志输出等级，彻底禁用网络连接日志及客户端心跳记录（`--quiet`），防止系统持续记录日常网络保活日志而触发频繁盘写动作。
* **TS 代理日志清理**：
  * 清理视频代理 `video-proxy` 中每个 3~10 秒 TS 视频切片被拉取时的高频日志控制台输出，阻断 Docker 标准输出重定向带来的系统系统日志持续写入。

### 2.3 群晖资源与带宽优化 (Commit: 0b40f9b)
* **智能分片协议直连**：
  * 对 m3u8 切片进行智能重写，将大流量 TS 视频流由 Node 中转代理转换为直接向目标 CDN 分发地址请求，使群晖本地服务器的 CPU 负载和公网下行出口带宽开销接近于 0。
* **海报客户端 CDN 代理**：
  * 对 DoubanCard 图片拉取实施客户端 CDN 代解析缓存，减少向本地 `image-proxy` 的重复高频请求。
* **MongoDB 内存 Cache 强制限制**：
  * 配置 MongoDB 引擎的 `wiredTigerCacheSizeGB` 限制其内存最高占用为 256MB，释放群晖宿主机内存。
